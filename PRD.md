## 1. 产品概述

### 1.1 产品愿景

让每个人都能轻松记录生活中的变化，通过智能拍照和自动合成技术，将平凡的时刻转化为震撼的延时视频，留住时光的轨迹。

### 1.2 目标用户

| 用户类型 | 典型场景 | 核心需求 |
|---------|---------|---------|
| 健身爱好者 | 记录身材变化 | 固定角度、定期提醒、对比查看 |
| 新手父母 | 记录宝宝成长 | 简单操作、自动合成、分享便捷 |
| 装修业主 | 记录装修进度 | 长期记录、进度回溯、成果展示 |
| 植物爱好者 | 记录植物生长 | 定时拍照、快速预览、延时效果 |

### 1.3 核心价值

- **零门槛：** 无需专业设备，一部手机即可创作延时视频
- **自动化：** 智能提醒拍照，自动合成视频，省时省力
- **有温度：** 记录真实生活变化，承载情感记忆
- **易分享：** 一键生成短视频，方便社交传播

## 4. 功能需求详情

### 4.1 项目管理

#### 4.1.1 创建项目

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 项目名称 | String | 是 | 最多 20 字符，如"减肥记录"、"宝宝成长" |
| 拍照频率 | Enum | 是 | 选项：每天/每周/每两周/每月/自定义 |
| 提醒时间 | Time | 是 | 24 小时制，如 08:00 |
| 封面照片 | Image | 否 | 默认为第一张照片 |

**处理逻辑：**
1. 校验项目名称不为空
2. 创建项目记录，生成唯一项目 ID
3. 注册定时提醒任务
4. 跳转到项目详情页面

**输出：** 项目创建成功，返回首页并显示新项目卡片

#### 4.1.2 项目列表

**输入：** 无

**处理逻辑：**
1. 查询所有项目，按创建时间倒序
2. 计算每个项目的进度（已拍天数/预计天数）
3. 显示项目封面（首图或默认图）

**输出：** 项目卡片列表，包含：
- 项目名称
- 已拍照片数量
- 进度百分比
- 最后拍照时间

#### 4.1.3 项目详情

**输入：** 项目 ID

**处理逻辑：**
1. 查询项目基本信息
2. 查询该项目下所有照片，按时间排序
3. 计算项目进度

**输出：**
- 项目信息区域（名称、创建时间、拍照频率）
- 照片墙（网格展示，支持点击放大）
- 操作按钮：拍照、生成视频、删除项目

#### 4.1.4 删除项目

**输入：** 项目 ID

**处理逻辑：**
1. 二次确认弹窗
2. 删除项目记录
3. 删除关联的照片文件
4. 删除关联的视频文件
5. 取消定时提醒任务

**输出：** 删除成功，返回首页

---

### 4.2 拍照功能

#### 4.2.1 手动拍照

**输入：** 项目 ID

**处理逻辑：**
1. 打开相机界面（默认后置摄像头）
2. 用户拍照
3. 保存照片到项目目录，命名规则：`{project_id}_{timestamp}.jpg`
4. 记录拍照时间到数据库
5. 更新项目最后拍照时间

**输出：** 拍照成功提示，返回项目详情

#### 4.2.2 拍照提醒

**输入：** 项目 ID、提醒时间、频率

**处理逻辑：**
1. 根据频率计算下次提醒时间
2. 注册本地通知（使用系统通知服务）
3. 通知文案："{项目名称} 该拍照啦！记录今天的变化~"
4. 用户点击通知直接打开项目详情

**输出：** 定时通知，点击可直达拍照

---

### 4.3 视频合成

#### 4.3.1 生成延时视频

**输入：** 项目 ID

**处理逻辑：**
1. 查询项目下所有照片，按时间排序
2. 校验照片数量 ≥ 2 张
3. 使用 FFmpeg 合成视频：
   - 帧率：10 fps
   - 分辨率：保持原图比例，最大 1080p
   - 格式：MP4 (H.264 编码)
4. 显示处理进度（0-100%）
5. 保存视频到项目目录：`{project_id}_timelapse.mp4`
6. 更新项目视频生成时间

**输出：** 视频生成成功，自动跳转到预览页面

#### 4.3.2 视频预览

**输入：** 视频路径

**处理逻辑：**
1. 加载视频文件
2. 提供播放/暂停控制
3. 支持全屏播放

**输出：** 视频播放器界面

---

### 4.4 分享导出

#### 4.4.1 保存到相册

**输入：** 视频路径

**处理逻辑：**
1. 请求相册写入权限
2. 复制视频文件到系统相册
3. 提示保存成功

**输出：** 保存成功提示

#### 4.4.2 分享视频

**输入：** 视频路径

**处理逻辑：**
1. 调用系统分享接口
2. 支持微信、朋友圈、QQ 等
3. 分享文案默认："{项目名称} 延时摄影，用 TimeLapse Camera 记录变化"

**输出：** 系统分享面板

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 启动时间 | < 2 秒 | 冷启动到首页可交互 |
| 拍照响应 | < 1 秒 | 点击拍照到保存完成 |
| 视频生成 | < 30 秒 | 20 张照片合成 1080p 视频 |
| 列表加载 | < 500ms | 项目列表加载完成 |
| 内存占用 | < 200MB | 正常使用场景 |

### 5.2 兼容性需求

| 平台 | 最低版本 | 目标版本 |
|-----|---------|---------|
| iOS | iOS 13.0 | iOS 17.0+ |
| Android | Android 8.0 (API 26) | Android 13+ |

### 5.3 存储需求

- 照片存储：使用应用沙盒目录，按项目分类
- 视频存储：同照片目录，单独子文件夹
- 缓存清理：提供清理缓存功能，保留原片

### 5.4 权限需求

| 权限 | 用途 | 必要性 |
|-----|------|-------|
| 相机 | 拍照 | 必需 |
| 相册读取 | 选择封面 | 可选 |
| 相册写入 | 保存视频 | 必需 |
| 通知 | 拍照提醒 | 必需 |

### 5.5 离线支持

- 核心功能完全离线可用
- 无需网络即可拍照、合成、保存
- 分享功能需要网络

## 附录

### A. 数据模型

```
Project (项目)
- id: String (主键)
- name: String (项目名称)
- frequency: Enum (拍照频率)
- reminderTime: String (提醒时间 HH:mm)
- coverImage: String (封面图路径)
- createdAt: DateTime (创建时间)
- updatedAt: DateTime (更新时间)

Photo (照片)
- id: String (主键)
- projectId: String (外键)
- imagePath: String (照片路径)
- takenAt: DateTime (拍照时间)
- order: Int (排序序号)

Video (视频)
- id: String (主键)
- projectId: String (外键)
- videoPath: String (视频路径)
- duration: Int (时长秒)
- createdAt: DateTime (生成时间)
```

### B. 命名规范

- 项目 ID：`proj_{uuid}`
- 照片文件：`{project_id}_{timestamp}.jpg`
- 视频文件：`{project_id}_timelapse.mp4`

### C. 错误码定义

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| E001 | 照片数量不足 | 提示用户至少需要 2 张照片 |
| E002 | 存储空间不足 | 引导用户清理空间 |
| E003 | 相机权限未授权 | 跳转设置页授权 |
| E004 | 视频生成失败 | 重试或联系客服 |
